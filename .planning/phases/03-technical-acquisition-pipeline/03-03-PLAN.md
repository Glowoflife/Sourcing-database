  **File 1: src/acquisition/site-crawler.ts**

  Key design rules:
  - One fresh `PlaywrightCrawler` instance per call to `crawlManufacturerSite()` — never shared across concurrent BullMQ jobs
  - `maxRequestsPerCrawl: 5` enforces D-01 (homepage + up to 4 inner pages)
  - Link enqueuing happens ONLY in the HOMEPAGE handler — inner pages must NOT enqueue further links
  - Keyword pattern: `/product|about|company|catalogue|our.products/i` (D-01 — uses regex for case-insensitive matching on href)
  - Use `transformRequestFunction` for link filtering. If TypeScript reports this option is not available in the installed Crawlee version, fall back to: extract links with `page.$$eval('a[href]', anchors => anchors.map(a => a.href))`, filter with the keyword pattern, and call `crawler.addRequests(filteredUrls.slice(0, 4))` manually.

  ```typescript
  import { PlaywrightCrawler } from "crawlee";
  import { htmlToMarkdown } from "@/acquisition/html-to-markdown";
  import type { CrawledPage } from "@/acquisition/types";

  // D-01: keyword pattern for link discovery (per CONTEXT.md — Claude's discretion on exact list)
  const KEYWORD_PATTERN = /product|about|company|catalogue|our.products/i;

  /**
   * Crawl a manufacturer's homepage and follow up to 4 keyword-matched inner pages.
   * Returns one CrawledPage per successfully crawled URL.
   * Each call creates a fresh PlaywrightCrawler instance — never reuse across concurrent jobs.
   *
   * @param homepageUrl  The manufacturer's website homepage URL (from leads.url)
   * @returns            Array of CrawledPage objects (homepage first, then matched inner pages)
   */
  export async function crawlManufacturerSite(homepageUrl: string): Promise<CrawledPage[]> {
    const results: CrawledPage[] = [];

    const crawler = new PlaywrightCrawler({
      maxRequestsPerCrawl: 5,         // D-01: cap at 5 pages total (homepage + 4 inner)
      maxRequestRetries: 3,            // D-02: 3 Crawlee-level retries per page before failedRequestHandler
      requestHandlerTimeoutSecs: 60,
      launchContext: {
        launchOptions: { headless: true },
      },

      async requestHandler({ page, request, enqueueLinks }) {
        const html = await page.content();
        const url = request.loadedUrl ?? request.url;
        const label = (request.label ?? "HOMEPAGE") as string;

        const pageType = inferPageType(url, label);
        const markdown = htmlToMarkdown(html, url);
        results.push({ url, pageType, markdown });

        // Only enqueue links from the homepage — do NOT cascade link-following to inner pages
        if (label === "HOMEPAGE") {
          await enqueueLinks({
            selector: "a[href]",
            label: "PAGE",
            transformRequestFunction(req) {
              // Filter: only enqueue links whose href matches the keyword pattern
              if (KEYWORD_PATTERN.test(req.url)) return req;
              return false; // skip non-matching links
            },
          });
        }
      },

      failedRequestHandler({ request, log }) {
        // Crawlee calls this after maxRequestRetries exhausted for a given URL
        // D-02: permanent failure — caller (runAcquisitionJob) checks results for homepage
        log.error(`Request failed permanently: ${request.url}`);
      },
    });

    await crawler.run([{ url: homepageUrl, label: "HOMEPAGE" }]);
    return results;
  }

  function inferPageType(url: string, label: string): CrawledPage["pageType"] {
    if (label === "HOMEPAGE") return "homepage";
    if (/product|catalogue/i.test(url)) return "products";
    if (/about|company/i.test(url)) return "about";
    return "other";
  }
  ```

  **File 2: src/acquisition/index.ts**

  This is the BullMQ job handler. It orchestrates status transitions and coordinates the crawl + DB writes.

  Error handling logic (from RESEARCH.md Q9):
  - If the homepage was NOT crawled (results has no entry with pageType 'homepage'), throw — this triggers BullMQ retry and eventually sets lead to Errored.
  - If the homepage was crawled but some inner pages failed, that is partial success — set lead to Crawled and log the partial outcome.
  - Always re-throw in the catch block so BullMQ records the job as failed (without re-throw, BullMQ considers the job successful even if lead is set to Errored).

  ```typescript
  import { db } from "@/db/index";
  import { leads } from "@/db/schema";
  import { crawlManufacturerSite } from "@/acquisition/site-crawler";
  import { writePage, updateLeadStatus } from "@/acquisition/page-writer";
  import { logger } from "@/lib/logger";
  import { eq } from "drizzle-orm";

  export async function runAcquisitionJob({
    leadId,
    url,
  }: {
    leadId: number;
    url: string;
  }): Promise<void> {
    const startedAt = Date.now();

    // Step 1: Mark lead as Processing before crawl starts (D-05 status machine)
    await updateLeadStatus(leadId, "Processing");

    logger.info({
      stage: "acquire",
      status: "start",
      leadId,
      message: `Crawling ${url}`,
    });

    try {
      // Step 2: Crawl manufacturer site — may return empty array if homepage fails all retries
      const pages = await crawlManufacturerSite(url);

      // Step 3: If homepage was not crawled, treat as permanent failure
      // (Crawlee exhausted retries on the homepage URL itself)
      const homepageCrawled = pages.some((p) => p.pageType === "homepage");
      if (!homepageCrawled) {
        throw new Error(`Homepage not crawled for leadId=${leadId} url=${url}`);
      }

      // Step 4: Write one manufacturer_pages row per crawled page (D-08)
      for (const p of pages) {
        await writePage(leadId, p);
      }

      // Step 5: Mark lead as Crawled on success
      await updateLeadStatus(leadId, "Crawled");

      const durationMs = Date.now() - startedAt;
      logger.info({
        stage: "acquire",
        status: "ok",
        leadId,
        durationMs,
        message: `Crawled ${pages.length} pages`,
      });
    } catch (err) {
      // Step 6: Mark lead as Errored on permanent failure (D-02)
      await updateLeadStatus(leadId, "Errored");

      logger.error({
        stage: "acquire",
        status: "fail",
        leadId,
        message: `Errored: ${String(err)}`,
      });

      // MUST re-throw so BullMQ records the job as failed and does NOT mark it completed
      throw err;
    }
  }
  ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BullMQ job data → runAcquisitionJob | Job payload {leadId, url} arrives from Redis; url is untrusted external input |
| Manufacturer website HTML → htmlToMarkdown → DB insert | Untrusted HTML content flows through Readability+Turndown into manufacturer_pages.markdown_content |
| Playwright browser → manufacturer website | Outbound browser request to external untrusted domain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | leadId from BullMQ job payload used directly in DB writes without validation | mitigate | runAcquisitionJob receives leadId as a number; Drizzle ORM uses parameterized queries for all updates and inserts. No SQL injection vector. AcquisitionJobSchema (Zod) can be used to validate job payload at the worker boundary — added as a discretionary improvement in acquisition.worker.ts (Plan 03-04). |
| T-03-08 | Tampering | Malicious manufacturer URL redirects to an unintended domain (open redirect) | accept | Crawlee's default strategy is same-hostname for enqueueLinks — inner page links are filtered to the same domain. Homepage URL originates from the leads table (written by the Chemexcil scraper from a public directory). Risk is low. |
| T-03-09 | Denial of Service | Manufacturer site returns infinite redirect loop or extremely slow response | mitigate | requestHandlerTimeoutSecs: 60 enforces a hard per-page timeout. maxRequestRetries: 3 caps retry attempts. After 3 retries, failedRequestHandler fires and the lead is set to Errored. |
| T-03-10 | Repudiation | Lead status set to Errored but BullMQ job marked as completed (missing re-throw) | mitigate | index.ts catch block re-throws the error after setting status to Errored, ensuring BullMQ records job failure. Acceptance criteria verifies `grep -c "throw err"` >= 1. |
