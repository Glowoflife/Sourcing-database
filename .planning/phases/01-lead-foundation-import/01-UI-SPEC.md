## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding (horizontal) |
| sm | 8px | Button internal padding, compact cell padding |
| md | 16px | Default table cell padding, form group gap, card padding |
| lg | 24px | Section padding, header vertical padding |
| xl | 32px | Main content horizontal padding |
| 2xl | 48px | Page-level top/bottom padding |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:
- Import result feedback card: 12px vertical padding (between sm and md — one-off for compact inline feedback, not a new token)
- Touch/click targets on the Import button: minimum 44px height

| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |

Notes:
- Body (14px/400/1.5): table cell text, import result counts, form labels

## Color

This is an internal procurement tool. The palette is neutral-dominant (professional, data-dense) with a single blue accent.

| Role | Tailwind Token | Usage |
|------|---------------|-------|
| Dominant (60%) | `background` / `zinc-950` dark or `white` light | Page background, table background |
| Secondary (30%) | `card` / `zinc-100` (light) or `zinc-900` (dark) | Page header bar, sidebar nav shell, import card container |
| Accent (10%) | `primary` / `blue-600` | Import button fill, active nav item indicator |
| Destructive | `destructive` / `red-600` | Error state text, Errored badge background |

shadcn CSS variables map directly: `--background`, `--card`, `--primary`, `--destructive` — use these, not hardcoded hex values. Tailwind v4 reads them via `@theme`.

**Accent reserved for:**
1. The "Import Leads" primary CTA button (background fill)
2. Active/current nav link indicator (left border or background tint)
3. Link URLs in the Name column (if made clickable in a later phase — do NOT apply in Phase 1 without explicit requirement)

**Status badge color contract (semantic, not accent):**

| Status | Badge Background | Badge Text | Tailwind Approach |
|--------|-----------------|------------|-------------------|
| New | `blue-100` | `blue-700` | `bg-blue-100 text-blue-700` |
| Processing | `yellow-100` | `yellow-800` | `bg-yellow-100 text-yellow-800` |
| Crawled | `green-100` | `green-700` | `bg-green-100 text-green-700` |
| Errored | `red-100` | `red-700` | `bg-red-100 text-red-700` |

Use shadcn `Badge` variant="outline" as the base, then override colors with these Tailwind classes. Do not use the default `Badge` color variants — status colors must be explicit.

## Component Inventory

### 1. CSV Import Section

**Location:** Top of `/leads` page, above the table. Not a separate route.

**Structure:**
```
<section aria-label="Import leads">
  <h2>Import Leads</h2>           — Heading (20px/600)
  <p>Upload a CSV file with columns: name, url</p>  — Body helper text (14px/400)
  <input type="file" accept=".csv" />               — Native file input (no custom drag-drop in Phase 1)
  <Button type="submit">Import Leads</Button>        — Primary accent button, 44px min height
  <ImportResultFeedback />                           — Shown after POST response only; hidden initially
</section>
```

**ImportResultFeedback (inline, not a modal):**
- Success: `{inserted} leads imported. {skipped} duplicates skipped.`
- Partial (some row errors): `{inserted} imported, {skipped} skipped. {errorCount} rows had errors — check that each row has a valid name and URL.`
- Error (no file / parse failure): `Import failed. {error message from API}.`
- Implemented as a conditional `<div role="status" aria-live="polite">` — not a toast, not a modal
- Rendered below the button; disappears on the next file selection

**Interaction flow:**
1. User selects a `.csv` file — button becomes enabled, file name shown adjacent to input
2. User clicks "Import Leads" — button shows loading state (`disabled + spinner icon`) during POST
3. Response received — ImportResultFeedback renders with result copy
4. Table re-fetches (Server Component reload or router.refresh())

### 2. Lead Status Table

**Location:** Below the import section on the same `/leads` page.

**Columns (in order):**

| Column | Width | Alignment | Component |
|--------|-------|-----------|-----------|
| Name | auto (flex 1) | left | plain text, `TableCell` |
| URL | auto (flex 1) | left | plain text (not a link in Phase 1) |
| Status | 120px fixed | center | `StatusBadge` (see badge spec above) |
| Added | 160px fixed | left | formatted date: `DD MMM YYYY` (e.g. "27 Apr 2026") |

**Empty state:** When 0 rows exist, render inside `<TableBody>`:
```
<TableRow>
  <TableCell colSpan={4}>
    <div class="text-center py-12">
      <p class="text-sm font-medium text-zinc-500">No leads yet</p>
      <p class="text-sm text-zinc-400">Import a CSV above to add manufacturers.</p>
    </div>
  </TableCell>
</TableRow>
```

**Ordering:** Newest first (`ORDER BY created_at DESC`) — no client-side sort in Phase 1.

**Pagination:** None in Phase 1. All rows rendered (acceptable for seed dataset size; Phase 5 adds virtualization).

### 3. Navigation Shell

**Location:** Wraps all dashboard routes via `/app/(dashboard)/layout.tsx`.

**Structure (sidebar layout):**
```
<aside aria-label="Main navigation" class="w-56 shrink-0 border-r">
  <nav>
    <a href="/leads" aria-current="page">Leads</a>    — only active nav item in Phase 1
  </nav>
</aside>
<main class="flex-1 overflow-auto px-8 py-12">
  {children}
</main>
```

**Phase 1 nav items:** Only "Leads" is active. Later phases add "Settings", "Discovery", "Pipeline", "Search", "Workflow". Stub links may be added as `aria-disabled` anchors with `opacity-40` to communicate future intent — do not add them unless the executor deems it valuable.

## Interaction States

| Element | States |
|---------|--------|
| Import button | default, hover, focus-visible (ring), loading (disabled + spinner), disabled (no file selected) |
| File input | idle, file-selected (show filename), error (browser native validation) |
| Table row | default (no hover highlight in Phase 1 — no row-level actions yet) |
| Status badge | static display only — no hover/click behavior in Phase 1 |
| Nav link | default, hover, active/current (accent left border + background tint) |

**Focus management:** After import completes, move focus to the ImportResultFeedback `<div>` via `focus()` so screen reader users hear the result without manual navigation.

**Loading feedback:** The Import button must enter its loading state immediately on click (before the API responds). Maximum perceived wait without visual feedback: 300ms — the button spinner covers this.

## Accessibility Contract

- All interactive elements must have visible focus rings (`focus-visible:ring-2 ring-offset-2`)
- Status badges must not convey meaning by color alone — the text label ("New", "Crawled", etc.) is the primary signal; color is secondary
- File input must have an explicit `<label>` association (`htmlFor` / `id` pairing) — not `aria-label` alone
- Import result feedback rendered in `<div role="status" aria-live="polite">` so assistive technology announces the result without requiring focus change
- Table must use correct semantic elements: `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th scope="col">`, `<td>` — shadcn Table component handles this
- Page heading hierarchy: `<h1>` = "Leads" (page title), `<h2>` = "Import Leads" (section heading)
