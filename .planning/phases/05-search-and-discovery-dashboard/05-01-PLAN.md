  <name>Implement Data Access Layer for Manufacturers</name>
  <files>src/lib/queries/manufacturers.ts</files>
  <behavior>
    - `getManufacturers({ q, industry, status, location, capacity, page })`:
      - Search (`q`): Match ILIKE against leads.name, leads.url, products.name, products.casNumber, locations.city, locations.state, locations.country, and manufacturerProfiles.industriesServed (array to text cast for ILIKE).
      - Industry Filter: array overlap (`&&`) for industriesServed.
      - Status Filter: match against leads.status (multi-select support).
      - Location Filter: match against locations.state (multi-select support).
      - Capacity Filter: bucketed logic for capacityMtPerYear (Unknown, &lt;100, 100-999, 1000-9999, 10000+).
      - Pagination: 50 rows per page, calculated using `limit(50)` and `offset((page - 1) * 50)`.
    - `getManufacturerDetail(leadId)`: Return lead + profile + products + locations + contacts.
  </behavior>
  <action>
    Create `src/lib/queries/manufacturers.ts`.
    Implement `getManufacturers` using Drizzle `db.select().from(leads).leftJoin(manufacturerProfiles, ...).leftJoin(products, ...).leftJoin(locations, ...)`.
    Use `and(...)` and `or(...)` from `drizzle-orm` to build the complex filter/search query.
    For search `q`, use `ilike` on multiple columns wrapped in `or`. For `industriesServed`, cast the text array to text for search.
    Implement pagination using `limit` and `offset` based on the `page` parameter (default to 1).
    For Capacity, implement the buckets using `gte` and `lt` filters or custom SQL CASE logic if doing it purely in DB.
    Implement `getManufacturerDetail` fetching all relations for the given `leadId`.
  </action>
